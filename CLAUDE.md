# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

gbrain-deploy is a deployment toolkit for [gbrain](https://github.com/garrytan/gbrain) — a personal/company AI knowledge brain by Garry Tan (Y Combinator). This project provides a unified CLI for self-hosting gbrain as an MCP (Model Context Protocol) server, allowing multiple AI agents (Claude Code, Cursor, Hermes, OpenClaw, etc.) to share the same knowledge base.

Docker-only deployment: single Docker Compose stack. Two network exposure modes: `public` (Caddy + Let's Encrypt HTTPS, requires domain) or `private` (plain HTTP bound to `GBRAIN_BIND_ADDR`, for Tailscale/LAN/local).

## Common Commands

```bash
# Syntax-check all scripts
for f in gbrain.sh lib/*.sh cmd/*.sh scripts/entrypoint.sh; do bash -n "$f"; done

# Deploy (interactive wizard)
./gbrain.sh deploy

# Check service status
./gbrain.sh status

# View logs
./gbrain.sh logs -f

# Register a new agent (issues a one-time gbrain_<hex> token)
./gbrain.sh agents add <name>

# List registered agents
./gbrain.sh agents list

# Backup (AES-256 encrypted .tar.enc, passphrase from BACKUP_PASSPHRASE in .env)
./gbrain.sh backup

# Restore from backup (requires same BACKUP_PASSPHRASE)
./gbrain.sh restore backups/gbrain-<timestamp>.tar.enc

# Run smoke tests
./gbrain.sh test

# View/edit configuration
./gbrain.sh config view
./gbrain.sh config set GBRAIN_PORT 3001

# Service management
./gbrain.sh start
./gbrain.sh stop
./gbrain.sh restart

# View Docker logs
docker compose logs -f gbrain
```

## Architecture

```
gbrain.sh (unified CLI entry point)
    │
    ├── cmd/deploy.sh — Interactive Docker setup (public domain or private/Tailscale)
    │   ├── .env → docker-compose.yml → Dockerfile → entrypoint.sh → gbrain serve
    │   ├── pgvector/pgvector:pg16 (embeddings DB)
    │   ├── ollama/ollama (optional, profile-gated)
    │   └── caddy (optional, profile-gated, public mode only)
    │
    ├── cmd/status.sh — Service health check
    ├── cmd/logs.sh — Docker log viewing
    ├── cmd/agents.sh — Agent registration and management
    ├── cmd/backup.sh — Backup database + brain data
    ├── cmd/restore.sh — Restore from backup
    ├── cmd/config.sh — Configuration management
    ├── cmd/service.sh — Start/stop/restart service
    └── cmd/test.sh — Smoke tests
```

**Data flow:** Agents → MCP endpoint (`/mcp`, HTTPS via Caddy in public mode or HTTP in private mode) → gbrain server → PostgreSQL (pgvector) + brain data directory (`/root/.gbrain`)

**Key components:**

- `lib/common.sh` — Shared functions (colors, prompts, config loading, health checks)
- `scripts/entrypoint.sh` — Docker container init: waits for Postgres, runs `gbrain init` (first run), `gbrain install` (skills), git repo init + remote pull, then `exec gbrain serve`
- `docker-compose.yml` — Three services: postgres, ollama (optional), gbrain. Volume `brain-data` persists brain content.
- `cmd/` — CLI command implementations

**Config flow:**

- `.env` in repo root (generated by `./gbrain.sh deploy`) → passed to compose and container
- `EXPOSE_MODE=public|private` in `.env` controls whether the Caddy compose profile is activated

**Environment variable groups:** Database (POSTGRES_*), Server (GBRAIN_PORT/GBRAIN_ADMIN_SECRET/GBRAIN_REF), Network exposure (EXPOSE_MODE/DOMAIN/ACME_EMAIL/GBRAIN_BIND_ADDR), Backup (BACKUP_PASSPHRASE/BACKUP_KEEP), LLM (LLM_API_BASE/KEY/MODEL), Embedding (OPENAI/ZeroEntropy/Voyage/Ollama), Git sync (BRAIN_GIT_REMOTE/BRANCH/TOKEN/USER/EMAIL).

## Important Conventions

- The project is bilingual: README.md and AGENT.md are in Chinese, doc/ contains English tutorials and Chinese translations
- Scripts use `set -euo pipefail` for safety
- Interactive prompts use helper functions: `prompt_text`, `prompt_password`, `prompt_select`, `prompt_yesno`
- Colors: RED=error, GREEN=success, YELLOW=warning, CYAN=info, BOLD=headers
- `gen_secret()` generates 32-char base64 tokens for passwords/admin secrets
- Admin bootstrap token must be 32+ chars matching `[A-Za-z0-9_-]` (admin use only, not an agent token)
- Agent tokens are `gbrain_<hex>` issued by `gbrain auth create` via `./gbrain.sh agents add <name>`; stored in `credentials/<name>.json`
- Git sync is opt-in (defaults to N) — supports GitHub PAT token auth or SSH

## What This Is NOT

This is not the gbrain source code. It's a deployment wrapper. The actual gbrain CLI, MCP server, and admin dashboard come from the upstream [garrytan/gbrain](https://github.com/garrytan/gbrain) repo, installed via `bun install -g github:garrytan/gbrain` (local) or cloned + built in the Dockerfile.
